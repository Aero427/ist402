Github is a popular space for developers in and out of open source software communities to collaborate. Microsoft recently purchased Github but the platform has been popular with developers of open source projects for over a decade prior.

Github has gained popularity in part because

  • Git as a version control system has largely won the popularity contest
  • They allow you to use the platform for free so long as what you do is open source
  • Private repos then allow companies to align their open contribution strategy with their internal development workflows
  • Github's social networking aspects around issue queues, following, forking and staring repos have all helped improve the social aspect of software development online
  • Github is adding capabilities to make code contributions without needing to understand everything that goes into making the contribution. Capabilities like clicking edit on the website, making a change in the browser and typing the commit name, then it automatically saving the change or creating a "Pull Request" for whoever owns the file
  • All of these reasons are why developers and non-developers Â are joining Github to do a lot of open source contribution.
  • Lots of other services and "apps" are starting to integrate with Github as a platform and as a login system

There is an open source variation of GitHub called GitLab. Gitlab is used by the university to share code internally (like WebAccess, security related code), automate workflows and increase collaboration between organizations.

For the purposes of this class we're going to focus on Github as it will provide the most visibility for you and what you are showcasing as being able to do.
