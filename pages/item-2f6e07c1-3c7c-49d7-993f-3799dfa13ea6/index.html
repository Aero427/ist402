Scenario

  • There are websites on the internet that are not made up to snuff for accessibility and that makes people without screens mad
  • Beyond mad, it makes them unable to actually use your product / website
  • This is both bad to do to a human being and stupid; you want their cash regardless of how they use your website!
  • But if only there were ways of learning how to improve the accessibility and SEO of a website at the same time...

Deliverable

  • Perform a web.dev / lighthouse and WAVE audit on a website
  • Review the feedback it gives and identify three areas for improvement from the audit that were things you didn’t know about before
  • Research (Google) how to solve these things and propose potential solutions to the problems
  • These problems can be SEO / A11y / Performance related; anything that’s given by the reports that you can interpret, research and then write / do a video about how to resolve

Option 1

  • Audit a website for a club, small business or other website affiliated with you or important to you in some way
  • This could also be something you don’t care about, just has to be something that generates meaningful data to analyze
  • Write on dev.to or hax.psu.edu for this option
  • If you own the site and can fix it this would make for an even more powerful / actionable way to solve this

Option 2

  • Audit a specific page in the course website or using hax.psu.edu to make a basic hello-world page
  • Do your write up in our team’s issue queue https://github.com/elmsln/issues/issues
  • By doing this you’ll help our projects (which are big while team is small) and improve your own SEO footprint via github posting / cross posting
    • next week we’ll get into github so this is a look ahead

Submission

  • Create the post, link to you video in the post
  • Submit link to post in #lab4-a11y-seo channel on Slack
  • This is due by Sunday at 11:59pm

Rubric

  • Did you run a Lighthouse / web.dev / WAVE audit (20%)
  • Did you write / research 3 areas for improvement (40%)
  • Did you do a video explaining the process / how to use these tools and what they are (40%)
  • Evaluation is contextual based on which option you pick.
