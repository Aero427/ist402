How to submit to lab

For practice as this is a code concept we are going to be taking a small coding exercise. Don't worry though, Google's developers have you covered with a great, easy to understand 5 page tutorial built into their documentation.

  1. Complete the Try LitElement practice exercise.
  2. Pick one of these to frame your writing and video response:
    • Noncoders:  Make the business , security , accessibility  or design consistency  case for using web components . Why is this a technology your next employer should consider adopting? Who else is doing it and what does it mean for development? Inspect source on websites such as HAXTheWeb , YouTube , Comcast , EA Games or others in order to make the case real.
    • Coders:  Package manager / tooling pro : Setup yarn / npm and create an element using Open-WC, the tooling the pros use https://open-wc.org/guide/  and do a video on that process, making a simple hero-card with properties like name (String ), cape (Boolean ), super powers (Array ), things like that or make up your own element that makes sense to you like portfolio-card and demonstrate usage.
  3. Write up 250 words making the case for web components or explaining what you did in the video and how someone else can (in the case of doing the making a web component case).
  4. Add your write up and video to a post on Dev.to . Dev.to is a popular medium clone that's growing in popularity with developer audiences and there's a good chance others will read your article given that there's growing interest in web components! (you can post on a hax.psu.edu site if you don't feel comfortable doing this write up publicly).
  5. Post a link to your dev.to article and #lab-10-webcomponents  in slack
